Angelic Doctor Academy

Full Stack Web Application

I began working on this site early on when there was only just a handful of draft code. I built off that code and turned into a full fledged site for a homeschooling curriculum / academy, Angelic Doctor Academy. This site is the home of a remote, correspondence highschool that allows parents/guardians register students, and then pick and choose courses for them in a fully customized experience. It also involved desiging innovative software solutions, such as automatically generated lesson plans, dynamically calculated graphical grade reports, and other periodical reminders, all done through custom built software. Below are some of the specific tasks I accomplished:

  • Built out a complex registration system for parents to enroll students, both front and backend
  • Re-engineered database structure for various new features; wrote new procedures and views
  • Converted an extensive Perl script to PHP in order to integrate third-party Learndash LMS functionality with corporate LMS
  • Used Canvas.js to display charts of student academic progress over courses and lessons, and email parent's these progress charts
  • Used ImageMagick PHP library and miniPaint editor to create a web based image markup app for grading homework images
  • Developed an administrative plugin to handle managing new courses, books, and users
  • Develop a C# app that acted as a customized FTP server using socket connections
  • Set up synchronization of WooCommerce products with the academy's courses and books using the WooCommerce API
  • Automatically synchronize student enrollments with corresponding WooCommerce Products
  • Developed a lesson plan generator that gives day by day instructions for each course for each student; used C# automation with Word to format these into Word Documents

Erectors, INC

Job/Schedule organizer for a construction company.

In a two-week-long sprint, I worked on a team of fellow junior software developers in creating an ASP.NET web app for Erector's INC. The purpose of this site was to allow the administrators to create work schedules for their employees, in order to easily organize and keep track off the company's tasks. Below are a list of my contributions to the project:

1. Redisigning the Navbar. I looked at the existing sister site as the model for the styling. Using jquery to alter the opacity of the navbar upon scrolling.

2. Creating a chatbox (front end). Learned about textarea tag in HTML, how to create a partial view layered on an existing view and common/visible on specific pages, the importance of referencing JS/Jquery tags in the right order. I styled the box similarly to the navbar, and made it collapsible. The heading had the name of the person to whom the message was being sent. Then I added a dropdown list from which the user could choose who to send the message to. Since the chatbox was supposed to be accesible on different pages, I could only pass a model to it from a controller that was common to the various controllers for the different pages. So I created a base controller from which the others inherited, and in that controller I queried the database for all the site users who would potentially become recipients for the messages.

3. Saving the messages to the DB. On the server side chathub class, I was able to retreive the necessary data and save to the DB.

4. Creating a code-first model for Entity Framework with foreign keys. (no code snippet)

5. Making a controller to get list of schedules from DB and organizing them according to jobs, then creating a view page where this data can be displayed in a tabular form.
